# 项目完成总结

更新时间：2026-04-06
工作目录：`/root/Dashboard`

## 1. 项目目标与总览
本项目已完成从“数据采集 -> 入库 -> 预测 -> 可视化大屏”的端到端闭环：
- 通过 MQTT 实时接收 6 个区域传感器数据。
- 使用 FastAPI 提供数据与预测查询接口。
- 将实时数据写入 MySQL `Real` 表，并进行整点时间归整。
- 独立预测脚本每小时生成未来 24 小时预测，写入 `predictions` 表。
- 前端科技大屏支持区域切换、实时数据展示、24h 预测图与 YOLO 占位展示。

## 4. 数据库与写入策略
### `Real` 表
- 保存实时传感器宽表数据。
- 时间字段按整点归整，便于按小时统计与建模。

### `predictions` 表
- 保存每个区域未来 24 小时预测结果。
- 唯一键：`(zone_id, predict_time)`。
- 每次整点运行对同一预测时刻执行覆盖更新，避免重复记录。
